# 90 Days DevOps & Cloud Learning Plan

## Current Level
I am currently working as a Quality Analyst at Amazon and transitioning towards DevOps & Cloud Engineering.  
I have basic technical understanding but limited real-world DevOps experience.  
For the next 90 days, my focus is consistency, hands-on practice, and building real skills instead of only watching tutorials.

---

## My Understanding of DevOps & Cloud
DevOps is the practice of building, deploying, automating, and managing applications faster and more reliably using tools, collaboration, and automation.

Cloud Engineering is about running applications and infrastructure on cloud platforms like AWS using scalable and modern infrastructure.

---

## Why I Am Learning DevOps
- I want to move from repetitive work to high-impact technical work.
- I want to build real-world skills that are in demand.
- I want a career with better growth, salary, and technical depth.
- I want to become someone who can build, deploy, and manage complete systems confidently.

---

## My 90-Day Goals
1. Build strong Linux, Git, Docker, and networking fundamentals.
2. Learn CI/CD, Kubernetes, and AWS basics through hands-on projects.
3. Complete the 90DaysOfDevOps challenge consistently and build a public GitHub portfolio.

---

## Core Skills I Want to Build
- Linux troubleshooting and command-line confidence
- CI/CD pipeline understanding and automation
- Docker and Kubernetes deployment/debugging

---

## Weekly Time Commitment
- Monday to Friday: 2–3 hours daily
- Saturday & Sunday: 5–6 hours daily including live classes and practice

---

## My Consistency Plan
- Study every single day even if progress feels slow.
- Push daily updates to GitHub to track progress publicly.
- Focus on understanding instead of rushing through topics.
- Avoid tutorial overload and spend more time practicing.
- Complete one task at a time with discipline for the next 90 days.

---

## Final Goal
By the end of these 90 days, I want to confidently build, deploy, and manage applications using DevOps tools and cloud platforms and become ready for DevOps-focused opportunities.
